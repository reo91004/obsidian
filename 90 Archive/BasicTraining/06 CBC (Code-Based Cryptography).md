# 4️⃣ Attacks on Code-based Cryptography

코드 기반 암호의 일반적인 공격 문제는 주어진 코드를 디코딩할 수 있는 디코더를 설계하는 것으로 귀결된다. 이는 주어진 신드롬에서 희소한 오류를 찾는 문제와 동일하다.

## [[Syndrome Decoding Problem (SDP)]]

-   **SDP의 정의**: 주어진 패리티 체크 행렬 $H$, 신드롬 $s$, 그리고 오류 가중치 $w$에 대해, $He = s$를 만족하며 $wt(e) = w$인 오류 벡터 $e$를 찾는 문제이다.
-   **용어 설명**:
    -   $H$: 코드워드가 반드시 만족해야 할 선형 규칙들의 묶음 (패리티 체크 행렬). 공격자는 랜덤 코드를 가정하므로 랜덤 패리티가 주어진다.
    -   $s$: 신드롬. 오류 그 자체가 아닌, 오류에 대한 선형 측정값이다.
    -   $e$: 오류 벡터. 암호 시스템에서 메시지나 키가 될 수 있다.
    -   $w$: 오류 벡터 $e$의 해밍 가중치 ($wt(e)$). 즉, $e$에서 1의 개수.
-   **난이도**: 선형 방정식 $He = s$만 보면 미지수 $n$과 방정식 $r$로 인해 해가 무수히 많다. 하지만 $wt(e) = w$라는 조건을 추가함으로써 "조합적으로 가장 적은 1을 가진 해를 찾는 문제"가 되며, 이는 **NP-hard** 문제이다.
-   **희소한 오류**: '희소하다'는 것은 '약하다'는 의미가 아니라 '조합적으로 적다'는 것을 의미한다. 즉, 전체 비트 중 $w$개만 뒤집힌 오류의 위치를 찾는 것이다.

## 공격 전략

코드 기반 암호의 일반적인 공격 문제는 주어진 코드를 디코딩할 수 있는 디코더를 설계하는 것으로 귀결된다. 이는 주어진 신드롬에서 희소한 오류를 찾는 문제와 동일하다.

### 일반적인 공격 (Generic Attacks)

-   `Goppa Code`든 `Random Code`든 공격자가 보는 문제는 동일하다.
-   `Information Set Decoding (ISD)`: $H$에 특별한 구조가 없다는 전제 하에 가우스 소거법 등을 사용하여 $e$를 찾으려 시도한다.

### 특정 코드에 대한 공격

-   `Goppa Code`: `오류 위치 다항식`, `유한체 구조`, `Patterson 알고리즘` 등을 사용하여 다항 시간 내에 해결 가능하다.

### ISD 계열 개선: Prange → Lee–Brickell → Leon → Stern

ISD 계열은 랜덤 코드를 대상으로 하는 일반적인 공격 방법이며, 핵심은 주어진 신드롬 $s$와 패리티 체크 행렬 $H$ ($r \times n$ 차원)에서 해밍 가중치 $w$를 가진 오류 벡터 $e$를 찾는 것이다. 이들 알고리즘은 가우스 소거법을 통해 $H$를 $[P \mid I_r]$ 형태로 변환하여 동치 문제로 만든 후 오류 $e$를 탐색한다.

-   **Prange**:
    -   **상황 설정**: $H \in \mathbb{F}_2^{r\times n}$ 이고 $r=n-k$일 때, 컬럼 치환(permutation)과 행 연산(row operation)을 통해 $H$를 $[P \mid I_r]$ 형태로 변환한다.
    -   **핵심 아이디어**: 오류가 항등 행렬 $I_r$ 쪽에만 집중되어 있다($e_P=0$)고 가정하여, 신드롬 $s$ 자체가 오류 벡터 $e_{I_r}$가 되므로 이를 즉시 찾는다. 이 가정이 맞을 때까지 반복 시도한다.
-   **Lee–Brickell (리브리겔)**:
    -   **핵심 변화**: Prange처럼 오류가 $I_r$ 쪽에만 있다고 고정하지 않고, **일부 오류가 왼쪽 $P$ 파트(정보 비트 쪽)에도 있을 수 있음**을 허용한다.
    -   **오류 분할 가정**: 왼쪽 $P$ 파트(컬럼 수 $n-r$)에 $p$개의 오류, 오른쪽 $I_r$ 파트(컬럼 수 $r$)에 $w-p$개의 오류가 있다고 가정한다.
    -   **아이디어**:
        1.  왼쪽 $P$ 파트에서 $p$개의 컬럼을 "찍어서" 더한다(선택된 컬럼에 해당하는 오류 효과를 신드롬에서 상쇄시키려는 시도). 바이너리에서 덧셈은 뺄셈과 동일하므로, $p$개의 오류가 기여하는 부분을 신드롬에서 제거한다.
        2.  제거 후 남는 신드롬 $s'$가 **오른쪽 $I_r$에 의해 설명 가능한지**($s'$의 해밍 가중치가 $w-p$인지 등) 체크한다.
        3.  만약 맞다면, $I_r$의 비영(1) 위치가 남은 오류 위치가 되며, 이를 바탕으로 치환된 원래의 오류를 복원한다.
    -   **체크 조건**: 후보 집합 $J \subseteq \{1,\dots,n-r\}$ ($|J|=p$)를 골라 $s' = s + \sum_{j\in J} c_j$ (여기서 $c_j$는 $P$ 파트의 컬럼)를 만들고, $s'$가 $I_r$만으로 설명되는지 검사한다.
    -   **복잡도 직관**: "왼쪽에 $p$, 오른쪽에 $w-p$"인 오류 패턴을 한 번 시도에서 맞출 확률은 대략 $\frac{\binom{n-r}{p}\binom{r}{w-p}}{\binom{n}{w}}$이며, 기대 반복 횟수는 이 확률의 역수이다.
    -   **트레이드오프**: $p$를 키우면 "성공 사건"의 범위가 넓어져 기대 반복 횟수는 줄어들지만, 왼쪽에서 $p$개의 컬럼을 고르는 조합 $\binom{n-r}{p}$이 증가하여s**내부 연산량이 증가**한다. 실무적으로 $p=2$와 같은 작은 값이 많이 사용된다.
-   **Leon**:
    -   **추가 가정**: Lee–Brickell처럼 $p$개 오류를 처리하되, 추가로 **신드롬의 앞 $L$비트가 전부 0**("여기에는 오류가 절대 없다")과 같은 특정 패턴을 가정한다.
    -   **장점**: 전체 해밍 가중치를 끝까지 확인하기 전에, **앞 $L$비트만 보고 0이 아니면 즉시 버릴 수 있어** 테스트가 빨라진다.
    -   **효과**: "맞아야 하는 조건"이 강해져 반복 횟수가 늘 수 있지만, 한 번의 후보 검사 비용을 훨씬 줄여 전체 기대 비용을 줄이는 방향으로 설계가 가능하다.
-   **Stern (1989)**:
    -   **한 단계 확장**: Leon의 "앞 $L$비트 0" 조건을 유지하면서, 왼쪽 파트($n-r$ 컬럼)를 **둘로 나누고**, $p$개를 고르는 일을 **Meet-in-the-Middle 형태로 분할**한다.
    -   즉, $p$개를 한 번에 고르는 대신 절반에서 $p/2$개, 나머지 절반에서 $p/2$개를 골라 합이 조건을 만족하는지 보며 **탐색 공간을 줄이는** 방식이다.
    -   **결론**: Prange(초기) 대비 Lee–Brickell → Leon → Stern으로 오면서, 지수 복잡도 $2^{\alpha n}$에서 $\alpha$가 점진적으로 낮아지는 형태로 개선이 축적되었다. 이는 획기적인 급락이라기보다는 $\alpha$가 조금씩 줄어드는 안정적인 개선이며, 따라서 $n$을 조금 늘리면 동일 보안 레벨 유지가 가능하다는 메시지를 전달한다.

## 이 파트의 결론: ISD 계열 공격의 의미

-   ISD 계열(Prange/Lee–Brickell/Leon/Stern)은 **코드 기반 암호의 파라미터(특히 코드 길이 $n$, 오류 가중치 $w$, 패리티 체크 행렬 차원 $r$)를 정하는 기준점**이다.
-   이러한 공격 기법들이 개선되어도 $\alpha$ 값이 "조금" 줄어드는 정도이므로, 공격 성능 개선에 대응하여 **코드 길이를 늘려서 보안 레벨을 따라잡는 설계**가 가능하다는 관점을 강조한다. 이는 코드 기반 암호 시스템의 장기적인 보안 신뢰성에 대한 기반이 된다.

# PQC 코드 기반 KEM 후보: Classic McEliece / BIKE / HQC

코드 기반 암호 시스템은 NIST PQC 표준화 과정에서 중요한 KEM(Key Encapsulation Mechanism) 후보들로 논의되고 있다. 주요 후보는 Classic McEliece, BIKE, HQC이다.

## 1) Classic McEliece

-   **정체성**: 고전적인 McEliece/Niederreiter 암호 시스템을 현대적으로 재해석하고 KEM 형태로 설계한 것이다. 여러 초기 제안들이 통합되어 'Classic McEliece'로 정리되었다.
-   **타입**: Niederreiter 타입의 구조를 사용한다.
-   **기반 난제**: 기본적으로 **Syndrome Decoding 난제**에 기반하며, 특히 랜덤 코드의 디코딩 어려움을 가정한다.
-   **장단점**:
    -   **장점**: 역사적으로 가장 오래되고 안정적인 보안 신뢰도를 가진다. **암호문(ciphertext)이 매우 작다**는 특징이 있다.
    -   **단점**: **키 사이즈가 매우 큼** (특히 공개키), 이는 TLS 같은 환경에서 부담이 될 수 있다.
    -   **주의사항**: 같은 오류(에러 벡터)를 재사용할 경우 보안상 위험이 발생할 수 있다는 점에 유의해야 한다.

## 2) BIKE (Bit Flipping Key Encapsulation)

-   **기반 코드/구조**: QC-MDPC(Quasi-Cyclic Moderate Density Parity-Check) 코드류를 기반으로 한다. 여기서 **'Moderate Density'**는 Low-Density Parity-Check(LDPC) 코드와 일반적인 랜덤 코드(High-Density)의 중간 밀도를 의미하며, **부분 순환(quasi-cyclic)** 구조를 특징으로 한다.
-   **구조적 장점**: "한 줄을 Cyclic Shift하여 나머지 줄이 정해지는" 구조 덕분에 저장 공간과 키 사이즈를 절감할 수 있다.
-   **키 아이디어**:
    -   비밀키는 로우 덴시티(희소한) 다항식 $(h_0, h_1)$ 등으로 표현된다.
    -   공개키는 개념적으로 $h_0^{-1}h_1$ 꼴로 생성되며, $h_0$는 로우 덴시티지만 그 역원 $h_0^{-1}$은 거의 랜덤(하이 덴시티)이 되므로, 결과적으로 공개키는 하이 덴시티처럼 보인다.
    -   이것은 "하이 덴시티 공개키에서 로우 덴시티 비밀키를 복원하기 어렵다"는 직관에 기반하여 보안을 설명한다.
-   **복호(디코딩)**: 비밀키를 사용하여 로우 덴시티 패리티 체크 형태를 복원한 다음, **LDPC(Low-Density Parity-Check) 코드 디코딩 알고리즘(비트 플리핑 또는 메시지 패싱 계열)**을 적용하여 오류를 찾는다.
-   **디코딩 실패**: 디코딩 실패 확률이 존재하지만, 이는 충분히 작게 관리되며, 실패 여부가 공격자에게 누설되지 않도록 CCA2(Chosen-Ciphertext Attack 2) 구성(예: 실패 시 랜덤 키 출력)에서 처리된다.
-   **비트 플리핑 직관**: 패리티 체크 행렬 $H$가 sparse(희소)한 경우, 신드롬과 컬럼들의 상관관계를 분석하여 상관이 높은 컬럼을 "에러 후보"로 점진적으로 좁혀나가는 반복적인 방식으로 오류를 찾아낸다.

## 3) HQC (Hamming Quasi-Cyclic)

-   **강조 포인트**: McEliece류처럼 "구조 있는 코드를 숨기기 위해 $(S,P)$ 치환으로 난독화"하는 전략에서 발생할 수 있는 구조-복원 공격 가능성을 **근본적으로 회피하려는 설계**를 목표로 한다.
-   **구조**: 공개키 쪽은 그냥 **랜덤 코드**(랜덤 $H$)로 두고, 복호에 필요한 부분은 **별도의 오류정정 코드**(예: Reed-Solomon 또는 Reed-Muller 코드의 결합)를 사용해 처리하는 "두 코드 동시 사용" 방식이다.
-   **암복호 직관**:
    -   암호문은 $(U,V)$ 두 파트로 구성된다.
    -   수신자는 비밀키(로우 웨이트 벡터들)를 이용하여 $U$를 가공하여 $V$에서 "큰 잡음 성분(하이 웨이트 오류)"을 상쇄시킨다.
    -   상쇄 후 남는 잔여 오차가 오류정정 코드가 감당 가능한 수준(로우 웨이트)이 되면, 해당 오류정정 코드의 디코딩 알고리즘을 사용하여 메시지를 회복한다.
-   **요약**: 공격자는 랜덤 코드의 Syndrome Decoding Problem을 풀어야 하는 반면, 수신자는 비밀키를 통한 "상쇄(cancellation)" 과정을 거쳐 디코딩 가능 영역으로 문제를 끌고 온 뒤 강력한 오류정정 코드를 사용하여 디코딩한다.

## 마지막 비교

NIST PQC 표준화 과정에서 코드 기반 KEM 후보들의 주요 특성을 동일 보안 레벨(예: 128-bit 보안 강도)에서 비교하면 다음과 같다.

-   **Classic McEliece**:
    -   **공개키 사이즈가 매우 큼**.
    -   암호문(ciphertext)은 매우 작다.
-   **BIKE / HQC**:
    -   공개키 및 암호문 사이즈가 Classic McEliece보다는 작거나 더 균형적이다.
    -   격자 기반 암호(예: Kyber)보다는 대체로 키/암호문 사이즈가 "조금 더 큼"이라는 뉘앙스가 있다.
-   **성능 (키 생성, 캡슐화, 디캡슐화)**:
    -   구현 및 환경에 따라 다르지만, 전반적으로 실용적인 속도 범주에 속한다.
    -   McEliece는 특히 캡슐화(encapsulation)가 빠른 편이다.
-   **결론**: 코드 기반 KEM 후보들 중 **적어도 하나 이상은 NIST PQC 표준으로 채택될 가능성**이 높다는 것이 강의의 마무리 요지이다. 이는 코드 기반 암호가 양자 컴퓨터 시대에 중요한 역할을 할 것임을 시사한다.
