# HQC, Hamming Quasi-Cyclic

## 한 줄 정의

해밍 부호와 준순환(Quasi-Cyclic) 구조를 사용한 코드 기반 포스트 양자 KEM 알고리즘.

## 왜 중요한가

- **NIST PQC 추가 표준**: 격자 기반 ML-KEM과 함께 NIST가 선정한 KEM 표준으로, 수학적 다양성을 제공한다.
- **수십 년 된 이론 기반**: 코딩 이론의 Syndrome Decoding Problem은 1978년(McEliece) 이후 깨지지 않은 검증된 난제이다.
- **관점의 전환**: 코딩 이론은 오류를 고치는 것이지만, 암호학에서는 **"오류를 고칠 수 있는 능력(비밀 키)을 가진 사람만 메시지를 복구할 수 있게 만드는 것"**으로 관점을 바꾼다.

## 핵심 직관

HQC의 이름이 이 알고리즘의 정체성을 설명한다.

- **Hamming:** 해밍 거리(Hamming Distance) 기반의 오류 정정 부호를 사용한다.
- **Quasi-Cyclic (준순환):** 행렬의 첫 줄만 알면 나머지 줄은 비트를 한 칸씩 밀어서(Rotation) 만들 수 있는 구조를 가진다.

핵심 아이디어는 **두 종류의 오류 정정 코드(RS, RM 등)를 중첩**하여 보안성과 복호화 성공률을 높이고, **Quasi-Cyclic 구조**를 사용하여 키 사이즈를 획기적으로 줄인 데 있다.

### 코드 기반 암호의 원리

$s = eH^T$를 통해 오류 $e$를 찾는 과정에서, 코드 기반 암호는 $H$는 공개하되 이를 효율적으로 풀 수 있는 **구조적 비밀**을 숨기는 방식이다. 공격자에게는 $s$와 $H$만 주어지며, $H$가 랜덤해 보이기 때문에 $e$를 구하는 것이 수학적으로 매우 어렵다(Syndrome Decoding Problem). 반면, 비밀 키를 가진 수신자는 효율적인 디코딩 알고리즘을 사용해 $e$를 제거하고 메시지를 읽는다.

### KEM 인터페이스

데이터 자체를 암호화하는 것이 아니라, 데이터를 암호화할 **대칭키(Secret Key)를 안전하게 나눠 갖기 위한 메커니즘**이다.

## 형식적 정의 / 수식

### 연결된 코드 구조 (Concatenated Codes)

암호화 과정에서 보안을 위해 의도적으로 많은 잡음(Error)을 섞는다. 복호화 시 이 잡음을 제거해야 하는데, 하나의 코드만으로는 실패 확률이 존재한다. 이를 해결하기 위해 두 겹의 방어막을 친다.

- **내부 코드 (Inner Code):** 잡음의 대부분을 1차적으로 걸러낸다. (보통 빠르고 가벼운 코드 사용)
- **외부 코드 (Outer Code):** 내부 코드가 놓친 잔여 오류를 완벽하게 제거한다. (**RS(Reed-Solomon)** 이나 **RM(Reed-Muller)** 이 이 역할을 수행한다)
- **결과:** 매우 높은 확률로 복호화 실패 없이 키를 공유할 수 있다.

### 준순환 구조의 효율성 (Quasi-Cyclic Structure)

- **일반 행렬:** $n \times n$ 크기의 패리티 검사 행렬 $H$를 저장하려면 $n^2$비트가 필요하다. $n$이 수만 단위라면 키 크기가 수 MB에 달해 비효율적이다.
- **Quasi-Cyclic 행렬:** 첫 번째 행 $h_0$만 저장하면 된다. 두 번째 행은 $h_0$를 오른쪽으로 한 칸, 세 번째 행은 두 칸 미는 식(Shift)으로 정의된다.
    - **메모리 복잡도:** $O(n^2) \rightarrow O(n)$
    - 이 덕분에 HQC는 강력한 보안성을 유지하면서도 키 크기를 수 KB 단위로 줄일 수 있다.

## 예시

### HQC의 작동 흐름

1. **키 생성 (KeyGen):** 수신자는 $H \cdot x = s$ 형태의 관계를 만족하는 희소 벡터(Sparse Vector)들을 생성하고, 이를 QC 형태로 압축하여 공개 키로 배포한다.
2. **암호화 (Encapsulation - Sender):**
    - 송신자는 공유할 비밀($m$)을 준비한다.
    - 이 $m$을 **외부 코드(RS/RM)**로 인코딩하고, 다시 **내부 코드**로 인코딩한다.
    - 여기에 의도적인 에러 벡터 $e$를 추가하여 암호문 $u, v$를 만든다.
3. **복호화 (Decapsulation - Receiver):**
    - 수신자는 자신의 비밀 키를 이용해 암호문에서 $m$에 섞인 노이즈를 수학적으로 제거한다.
    - 1차로 내부 코드를 디코딩하고, 2차로 외부 코드(RS/RM)를 디코딩하여 깨끗한 $m$을 얻는다.

### 다른 PQC와의 비교

|**구분**|**HQC (Code-based)**|**Kyber/ML-KEM (Lattice-based)**|
|---|---|---|
|**기반 문제**|신드롬 디코딩 문제 (Coding Theory)|LWE 문제 (Lattice Theory)|
|**복호화 실패율**|존재하지만 매우 낮게 설계 (Concatenated Code 덕분)|존재함|
|**키 사이즈**|큼 (QC 구조로 많이 줄였으나 여전히 큼)|작음 (가장 효율적)|
|**구현 난이도**|쉬움 (비트 연산 위주, 상수 시간 구현 용이)|중간 (복잡한 수치 연산 필요)|
|**성숙도**|매우 높음 (수십 년 된 이론 기반)|상대적으로 최신 이론|

## 자주 헷갈리는 점

- **HQC ≠ McEliece**: 둘 다 코드 기반이지만, McEliece는 Goppa 코드를 사용하고 HQC는 준순환 구조 + 연결 코드를 사용한다. HQC가 키 크기 면에서 더 효율적이다.
- **"오류를 더한다"는 것은 약점이 아니다**: 의도적으로 추가한 오류가 보안의 원천이다. 비밀 키 없이는 이 오류를 제거할 수 없다.
- **복호화 실패 가능성**: 이론적으로 존재하지만, Concatenated Code 구조 덕분에 실패 확률이 $2^{-128}$ 이하로 설계된다.

## 요약 체크리스트

1. **HQC의 정의:** 해밍 부호와 준순환 구조를 사용한 포스트 양자 암호(KEM) 알고리즘이다.
2. **안전성 원천:** "랜덤한 선형 코드는 복호화하기 어렵다"는 코딩 이론의 난제에 기반한다.
3. **구조적 특징:**
    - **Concatenated Codes (RS, RM 등):** 두 번의 인코딩/디코딩을 거쳐 복호화 실패 확률을 0에 수렴시킨다.
    - **Quasi-Cyclic:** 행렬 전체를 저장하는 대신 첫 행만 저장하여 키 사이즈를 획기적으로 줄인다.
