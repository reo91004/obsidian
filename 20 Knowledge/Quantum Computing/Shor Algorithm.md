# Shor Algorithm

## 한 줄 정의

큰 정수 $N$을 다항 시간 $O((\log N)^3)$에 인수분해하는 양자 알고리즘.

## 왜 중요한가

- RSA 등 현대 공개 키 암호의 안전성은 큰 수의 인수분해가 어렵다는 가정에 기반한다.
- Shor 알고리즘은 이 가정을 무너뜨려, 충분한 큐비트를 가진 양자 컴퓨터가 현행 암호 체계를 깨뜨릴 수 있음을 보였다.
- 이로 인해 [[PQC]] (양자내성암호) 연구가 촉발되었다.

## 핵심 직관

### 그래서, 쇼어 알고리즘이 뭔 상관인데?

찾아본 바로는, ==주기성==때문에 그렇다 한다.

예를 들어, RSA를 보자.

$$
a^x\equiv b \pmod N
$$

여기서 우리는 $a$를 order가 최대한 큰 수 or 원시근으로 선정하며 $x$를 비밀 키로 둔다. $b$를 공개 키로 생각하며, $N$을 큰 소수로 사용한다.

$x$를 정방향 계산하여 $b$를 산출하는 건 쉽지만, 역으로 $x$를 찾는 건 쉽지 않다.

$$
\text{Given } N=pq,\ \text{find } p,q
$$

그래서, Shor 알고리즘은 이걸 그대로 풀지 않고 주기 문제로 바꾼다.

임의의 (a)를 골라서 함수

$$
f(x) = a^x \bmod N
$$

를 생각한다.

* $f(x)$는 **주기적**이다.
* 어떤 최소의 $r$이 존재해서 $f(x+r)=f(x)$, 즉 $a^r \equiv 1 \pmod N$로 만들며, 여기서 $r$이 바로 **주기**다.

### 근데 왜 주기만 알면 인수분해가 되는데?

$$
a^r \equiv 1 \pmod N
\Rightarrow
(a^{r/2}-1)(a^{r/2}+1)\equiv 0 \pmod N
$$

이기 때문에,

$$
\gcd(a^{r/2}\pm 1, N)
$$

가 되며, 둘 중 하나가 확률적으로 $p$ 또는 $q$가 된다. 뭐 그렇겠지..? 곱해서 $(a^{r/2}−1)(a^{r/2}+1)=k⋅N$ 가 될 테니까.

==물론 여기까지는 양자랑 아무 상관 없다.==

여기서 고전 컴퓨터는 패턴이 보이지 않기 때문에 ($N$때문에 비선형성이 됨) 주기라는 정보를 관측할 방법이 없다. 허나,

$$
\sum_{x=0}^{Q-1} |x\rangle |f(x)\rangle
$$

양자 컴퓨터에서는 이런 식으로, 모든 $x$에 대해 동시에 f$(x)$를 계산(병렬)한다.

여기서부터 이해가 잘 안 갈 수 있는데, 첫 번째 레지스터에 [[QFT]]를 적용한다.

- 시간 영역 → 주파수 영역
- 값 → 주기

로 변환함으로써, ==이 함수가 어떤 주기를 갖는지를 간섭 패턴으로 변환==한다.

## 형식적 정의 / 수식

### 알고리즘 단계

**입력:** 합성수 $N = pq$

1. **전처리 (고전):** $\gcd(a, N) = 1$인 임의의 $a$를 선택. 만약 $\gcd(a,N) \neq 1$이면 이미 인수를 찾은 것이다.

2. **양자 주기 찾기:**
   - $n = \lceil \log_2 N \rceil$으로 두고, 두 개의 양자 레지스터를 준비: $|0\rangle^{\otimes 2n}|0\rangle^{\otimes n}$
   - 첫 번째 레지스터에 Hadamard 적용하여 균등 중첩 생성
   - 모듈러 지수 함수 $U_f: |x\rangle|0\rangle \to |x\rangle|a^x \bmod N\rangle$ 적용
   - 첫 번째 레지스터에 **역 QFT** 적용
   - 첫 번째 레지스터 측정 → 값 $c$를 얻음

3. **후처리 (고전):** 연분수 전개(continued fraction expansion)로 $c/2^{2n}$에서 주기 $r$의 후보를 추출.

4. **검증:** $r$이 짝수이고 $a^{r/2} \not\equiv -1 \pmod{N}$이면:
$$p = \gcd(a^{r/2} - 1, N), \quad q = \gcd(a^{r/2} + 1, N)$$

5. 실패 시 다른 $a$를 선택하여 반복. 성공 확률은 $1 - 2^{1-k}$ 이상 ($k$: 소인수 개수).

## 예시

### $N = 15$ 인수분해

1. $a = 7$ 선택 ($\gcd(7, 15) = 1$)
2. $f(x) = 7^x \bmod 15$를 계산하면:
   - $f(0) = 1,\ f(1) = 7,\ f(2) = 4,\ f(3) = 13,\ f(4) = 1,\ \ldots$
   - 주기 $r = 4$
3. $r = 4$는 짝수이므로:
   - $a^{r/2} = 7^2 = 49$
   - $\gcd(49 - 1, 15) = \gcd(48, 15) = 3$
   - $\gcd(49 + 1, 15) = \gcd(50, 15) = 5$
4. 결과: $N = 15 = 3 \times 5$

## 자주 헷갈리는 점

- **양자 부분은 주기 찾기뿐이다**: 인수분해 자체는 고전적 수론이고, 양자 컴퓨터는 주기 $r$을 효율적으로 찾는 역할만 한다.
- **항상 성공하는 것은 아니다**: $r$이 홀수이거나 $a^{r/2} \equiv -1 \pmod{N}$이면 실패하지만, 반복하면 높은 확률로 성공한다.
- **현재 양자 컴퓨터로 RSA를 깨는 것은 불가능하다**: 실용적 크기의 RSA 키를 깨려면 수천 개의 논리적 큐비트가 필요하다.
